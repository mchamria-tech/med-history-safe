

## Add AI-Generated Insights with Medical Disclaimer

### Overview
After the lab parameters are extracted, a follow-up AI call will generate educational insights for out-of-range parameters. These insights will be displayed in a dedicated "Insights" section below the parameters table, with a prominent medical disclaimer.

---

### 1. Update Edge Functions (both `analyze-lab-report` and `analyze-lab-report-user`)

Add a **second AI call** after the parameter extraction succeeds, only if there are out-of-range parameters. This keeps the parameter extraction reliable (structured tool calling) while the insights are a separate free-text generation.

**Second AI call prompt:**
- System: "You are a medical education assistant. Given a list of out-of-range lab parameters, provide brief educational context about what each parameter measures and what abnormal values may commonly be associated with. Do NOT diagnose. Do NOT recommend treatment. Use phrases like 'commonly associated with', 'may indicate', 'often seen in'. Always remind that only a qualified healthcare provider can interpret results in clinical context."
- User: The list of out-of-range parameters with their values and statuses
- Response: Plain text (no tool calling), returned as an `insights` string field alongside `parameters`

**Updated response shape:**
```json
{
  "document_name": "...",
  "document_date": "...",
  "parameters": [...],
  "insights": "string of educational context (only present when out-of-range params exist)"
}
```

### 2. Update Patient Analytics Page (`src/pages/PatientLabAnalytics.tsx`)

Add below the parameters table:
- A medical disclaimer banner (using Alert component) with an info icon, always visible when results are shown
- An "Insights" card that renders the `insights` text (only when present)

**Disclaimer text:**
"This analysis is generated by AI for **educational purposes only**. It is not a medical diagnosis, and should not be used as a substitute for professional medical advice, diagnosis, or treatment. Always consult a qualified healthcare provider for interpretation of lab results and any health concerns."

### 3. Update Partner Analytics Page (`src/pages/partner/PartnerClientAnalytics.tsx`)

Same additions -- disclaimer banner and insights card below the results table.

### Technical Details

**Second AI call (added after line ~300 in both edge functions):**
- Only triggered when `parameters.filter(p => p.is_out_of_range).length > 0`
- Uses `google/gemini-3-flash-preview` (same model, fast and cheap)
- No tool calling -- plain `choices[0].message.content` response
- If the insights call fails, return parameters without insights (graceful degradation)
- The prompt explicitly instructs the AI to avoid diagnosis language and use hedging phrases

**UI components used:**
- `Alert`, `AlertTitle`, `AlertDescription` from `@/components/ui/alert` for the disclaimer
- `Info` icon from `lucide-react`
- The insights text will be rendered with `whitespace-pre-line` to preserve paragraph breaks
